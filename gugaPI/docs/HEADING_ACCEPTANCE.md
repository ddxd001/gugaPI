# 航向闭环硬件验收测试

本文档是 `CHASSIS_CONTROL_PLAN.md` 阶段二的硬件验收流程。航向源为 ICM-45686 陀螺仪 Z 轴 yaw 积分（非 GY931），闭环代码在 `heading.cpp`，IMU 采样在 `app_imu.cpp`。

## 0. 前置知识与安全约定

### 0.1 航向源

| 项目 | 值 |
| --- | --- |
| 传感器 | ICM-45686（SPI，PC7 片选） |
| 采样周期 | 10 ms（`app_imu` 任务） |
| yaw 单位 | 毫度（mdeg），范围 `-180000..180000` |
| yaw 积分方式 | 陀螺仪 Z 轴角速度 × 真实微秒间隔，保留余量防截断 |
| 零偏 | `imu_gyro_bias_z_mdps`（ConfigStore，默认 0） |
| 航向控制周期 | 50 ms（`heading` 任务） |

### 0.2 两种模式

| 模式 | Shell 命令 | 行为 |
| --- | --- | --- |
| HOLD（直行保持） | `heading hold <rpm>` | 锁定当前 yaw，`left = base - corr`，`right = base + corr` |
| TURN（相对转弯） | `heading turn <deg>` | 原地差速转向，走最短路径到目标角度 |

### 0.3 默认参数（ConfigStore v3）

| 参数 | 默认值 | 含义 |
| --- | --- | --- |
| `heading_kp` | 1000 | 1 RPM/deg（`corr = error_mdeg * kp / 1e6`） |
| `heading_max_correction_rpm` | 30 | HOLD 差速修正上限 |
| `heading_turn_max_rpm` | 60 | TURN 最大轮速 |
| `heading_turn_min_rpm` | 20 | TURN 最小轮速（克服摩擦） |
| `heading_tolerance_mdeg` | 3000 | 到位容差（3°） |
| `heading_settle_ms` | 300 | 到位后保持时间 |

### 0.4 安全机制

| 触发条件 | 动作 | 故障码 | 恢复方式 |
| --- | --- | --- | --- |
| IMU 数据无效或 `valid=false` | 立即停车 | `FAULT_SENSOR_LOST` | 仅 `reset` |
| 航向任务被饿死 > 200 ms | 立即停车 | `FAULT_SENSOR_LOST` | 仅 `reset` |
| HOLD 航向误差 > 90° | 立即停车 | `FAULT_SENSOR_LOST` | 仅 `reset` |
| TURN 超时 8 s | 立即停车 | `FAULT_DRIVER_TIMEOUT` | 仅 `reset` |
| 底盘 I2C 命令失败 | 立即停车 | 无（不锁故障） | 可直接重新启动 |
| 已有故障时运行 | 立即停车 | 不覆盖 | 仅 `reset` |

> **重要**：一旦触发 `FAULT_SENSOR_LOST` 或 `FAULT_DRIVER_TIMEOUT`，故障被锁存，只能通过 `reset` 复位 MCU 恢复。Shell 没有故障清除命令。测试中遇到故障停车时，先执行 `reset` 再继续。

### 0.5 方向约定（需实测确认）

代码中 `kYawSign = 1`（`heading.cpp:17`），假设"右轮更快 → yaw 增大"。TURN 模式正角度（`turn 90`）预期为左转。**实际方向取决于 IMU 安装方向和电机接线**，第一阶段架空测试必须确认。

### 0.6 测试串口

UART0，PB1/RX、PB0/TX，`115200 8N1`。所有命令通过 Shell 执行。

## 1. 测试前准备

### 1.1 硬件准备

- [ ] 小车架空（轮悬空，可自由旋转）
- [ ] 急停开关/断电开关就手可操作
- [ ] 调试串口连接稳定（COM 口已识别）
- [ ] 地面测试区域清空，至少 2 m × 2 m
- [ ] 电池电压充足（建议 > 7V，6 节 AA 镍氢）

### 1.2 上电自检

依次执行：

```text
imu sample
```

确认输出含 `yaw=X.XXX deg`，`valid` 为 true。如果提示 `no data`，先执行：

```text
imu icm init
imu sample
```

```text
imu oled on 100
```

在 OLED 上实时观察 yaw 值，手动缓慢左右转动车身，确认 yaw 连续变化且方向正确（右转 yaw 增大、左转 yaw 减小为默认约定）。

```text
motor ping
motor info
```

确认 `id=0xA5`、`fw=2`、通信正常。

```text
chassis stat
```

确认 `last=ok`，两轮 `act=0`。

```text
heading status
```

确认 `mode=idle`。

> 如果以上任一项异常，先排查再继续。IMU 不工作则航向闭环无法运行。

## 2. 架空方向验证（轮悬空）

> 目标：确认 yaw 方向、HOLD 修正方向、TURN 转向方向正确。全程架空，电机低速运转。

### 2.1 yaw 方向确认

```text
imu oled on 100
```

手动将车头对准一个固定参考方向，记录 OLED 上的 yaw 值。然后：

1. 手动缓慢向右转约 90°，观察 yaw 应**增大**。
2. 手动缓慢向左转约 90°，观察 yaw 应**减小**。
3. 连续转一圈，确认 yaw 在 `-180°/180°` 处平滑回绕，不出现 360° 跳变。

> 如果方向相反（右转 yaw 减小），需要修改 `heading.cpp` 中 `kYawSign = -1` 并重新编译。**不要在未确认方向的情况下进行地面测试。**

### 2.2 HOLD 修正方向确认

```text
heading hold 30
```

车轮应以 30 RPM 直行。观察 2~3 秒后：

```text
heading stop
```

**预期**：两轮同向同速转动，无明显差速。

手动将车身偏转约 30°（模拟行驶中跑偏），再次启动：

```text
heading hold 30
```

**预期**：车轮出现差速修正——车身偏右时左轮应更快（把车拉回左）；偏左时右轮应更快。

> 如果修正方向反了（越修越偏），说明 `kYawSign` 需要取反。执行 `heading stop`，修改 `kYawSign` 后重新编译。

### 2.3 TURN 转向方向确认

```text
heading turn 90
```

**预期**：原地左转（差速转向，左轮后退、右轮前进），约 1~2 秒后停在目标角度。

```text
heading status
```

确认 `mode=idle`（已到位停车），`error` 应 < 3°。

```text
heading turn -90
```

**预期**：原地右转，转完后回到起始方向。

```text
heading turn 180
```

**预期**：原地转 180°（走最短路径，方向取决于起始 yaw 位置）。

> 如果 `turn 90` 实际右转，说明 `kYawSign` 需要取反。和 HOLD 方向一起确认后再改。

### 2.4 架空方向验证结论

| 检查项 | 通过条件 |
| --- | --- |
| yaw 方向 | 右转 yaw 增大，左转 yaw 减小 |
| HOLD 修正 | 跑偏后差速修正方向正确（拉回目标航向） |
| TURN +90° | 左转 |
| TURN -90° | 右转 |
| TURN 180° | 走最短路径 |

> **全部通过后才能进行地面测试。** 如果 `kYawSign` 被修改，需重新编译烧录后再进入地面测试。

## 3. 地面直行保持测试（HOLD）

> 目标：验证地面负载下航向保持精度。每次测试后记录起始和结束 yaw。

### 3.1 测试流程

对每个目标速度执行：

1. 将车放在地面起点，对准固定参考方向。
2. 记录起始 yaw：
   ```text
   imu sample
   ```
3. 启动航向保持：
   ```text
   heading hold <rpm>
   ```
4. 行驶约 2~3 秒后手动停车：
   ```text
   heading stop
   ```
5. 记录结束 yaw：
   ```text
   imu sample
   ```
6. 计算航向误差 = 结束 yaw − 起始 yaw（取最短角度差）。

### 3.2 测试矩阵

| 编号 | 目标 RPM | 行驶时间 | 起始 yaw | 结束 yaw | 航向误差 | 通过 |
| --- | --- | --- | --- | --- | --- | --- |
| H1 | 30 | 3 s | | | | |
| H2 | 50 | 3 s | | | | |
| H3 | 80 | 3 s | | | | |
| H4 | 80 | 5 s | | | | |
| H5 | 150 | 3 s | | | | |

### 3.3 扰动恢复测试

以 80 RPM 直行时，用手轻推车身产生约 10°~20° 的横向偏转：

```text
heading hold 80
```

行驶 2 秒后轻推车身侧面，观察：

**预期**：车轮差速修正，车身在 1~2 秒内回正，不出现蛇形振荡。

> 如果振荡，降低 `heading_kp`（如 `param set heading_kp 500`）。如果修正太慢，提高 `heading_kp`（如 `param set heading_kp 1500`）。修改后 `param save` 持久化。

### 3.4 重复性测试

以 80 RPM 连续执行 10 次直行（每次 3 秒），记录每次航向误差：

| 次数 | 误差 | 次数 | 误差 |
| --- | --- | --- | --- |
| 1 | | 6 | |
| 2 | | 7 | |
| 3 | | 8 | |
| 4 | | 9 | |
| 5 | | 10 | |

### 3.5 验收标准

| 指标 | 初期目标 | 稳定目标 |
| --- | --- | --- |
| 单次航向误差 | ≤ 5° | ≤ 3° |
| 10 次最大误差 | ≤ 8° | ≤ 5° |
| 修正不持续饱和 | 是 | 是 |
| 无蛇形振荡 | 是 | 是 |
| 扰动后能回正 | 是 | 是 |

## 4. 地面转弯测试（TURN）

> 目标：验证相对角度转弯精度。每次记录目标角度和实际角度差。

### 4.1 测试流程

1. 车放在地面，记录起始 yaw（`imu sample`）。
2. 执行转弯：
   ```text
   heading turn <deg>
   ```
3. 等待 `heading status` 显示 `mode=idle`（到位停车）。
4. 记录结束 yaw，计算实际转角 = 结束 yaw − 起始 yaw（最短角度差）。
5. 误差 = 目标角度 − 实际转角。

### 4.2 测试矩阵

| 编号 | 目标角度 | 起始 yaw | 结束 yaw | 实际转角 | 误差 | 通过 |
| --- | --- | --- | --- | --- | --- | --- |
| T1 | +30° | | | | | |
| T2 | -30° | | | | | |
| T3 | +45° | | | | | |
| T4 | -45° | | | | | |
| T5 | +90° | | | | | |
| T6 | -90° | | | | | |
| T7 | +180° | | | | | |
| T8 | -180° | | | | | |

> 每个角度测试 3 次，取最大误差。T5/T6 之间和 T7/T8 之间应回到接近起始方向。

### 4.3 回绕点测试

将车身转到 yaw 接近 ±180° 位置，然后执行：

```text
heading turn 90
```

**预期**：正确处理 `-180°/180°` 回绕，不出现接近 360° 的错误旋转。转弯方向应走最短路径。

### 4.4 左右对称性

比较 T5（+90°）和 T6（-90°）的误差：

**验收**：左转和右转误差之差 ≤ 2°，不依赖单向补偿。

### 4.5 验收标准

| 指标 | 初期目标 | 稳定目标 |
| --- | --- | --- |
| 90° 转向误差 | ≤ 5° | ≤ 3° |
| 180° 转向误差 | ≤ 5° | ≤ 5° |
| 左右误差差 | ≤ 3° | ≤ 2° |
| 回绕点不跳变 | 是 | 是 |
| 所有失败路径最终停车 | 是 | 是 |

> 如果转弯超调（转过头后反复摆动），增大 `heading_settle_ms`（如 `param set heading_settle_ms 500`）。如果转弯不到位，增大 `heading_kp` 或减小 `heading_tolerance_mdeg`。

## 5. 安全与故障路径测试

> 目标：验证所有异常路径都能安全停车。

### 5.1 IMU 断连测试

```text
heading hold 80
```

行驶中拔掉 IMU SPI 排线（或松开排针）：

**预期**：立即停车，触发 `FAULT_SENSOR_LOST`。

恢复：重新插好排线，执行 `reset`，确认 `imu sample` 恢复正常。

### 5.2 HOLD 误差过大保护

此测试较难人为触发（需要 yaw 在 HOLD 中偏移 > 90°），可通过以下方式近似验证：

```text
heading hold 30
```

行驶中用手强制将车身旋转 > 90°：

**预期**：立即停车，触发 `FAULT_SENSOR_LOST`。

> 此测试可能因手动旋转速度和力度不同而有差异。核心确认点是：大角度偏移后停车，而非继续盲目修正。

### 5.3 TURN 超时保护

用手按住车身（阻止转弯），执行：

```text
heading turn 90
```

**预期**：8 秒后超时停车，触发 `FAULT_DRIVER_TIMEOUT`。

### 5.4 底盘通信故障

此测试在 `heading hold 80` 运行中拔掉 gugaPI→MotorDriver 的 I2C 线：

**预期**：底盘命令失败 → 立即停车（不锁故障）。MotorDriver 看门狗 1 秒后超时也会 coast。

> 此路径不设故障码，恢复通信后可直接重新启动 `heading hold`。

### 5.5 安全路径验收

| 检查项 | 通过条件 |
| --- | --- |
| IMU 断连 | 立即停车 + FAULT_SENSOR_LOST |
| HOLD 误差 > 90° | 立即停车 + FAULT_SENSOR_LOST |
| TURN 超时 | 8 s 内停车 + FAULT_DRIVER_TIMEOUT |
| I2C 断连 | 立即停车（不锁故障） |
| 故障后无法启动 | `heading hold` 返回 not-initialized |

### 7.3 已知问题

1. **±180° 边界方向歧义**：`heading turn -180` 实际左转（`ShortestAngleDiff` 将 -180° 解析为 +180°）。结果正确（180° 转向后方向相同），但转向方向与命令符号不一致。仅在 ±180° 边界出现，其他角度不受影响。
2. **陀螺仪零偏**：ICM-45686 Z 轴静态偏差约 30~823 mdps（随温度变化），当前 `imu_gyro_bias_z_mdps=0` 未补偿。短时间测试（< 5s）影响可忽略，长时间直行可能产生 0.1~0.8°/s 漂移。建议后续标定。
3. **TURN 架空超时**：架空测试时 TURN 无法到位（车身不旋转），8 秒后必然超时。这是预期行为，不是缺陷。
