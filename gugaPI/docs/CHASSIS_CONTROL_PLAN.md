# 小车控制开发计划

## 1. 文档目的

本文档用于跟踪电赛小车控制功能的设计、实现、测试和验收状态。
每完成一个实现步骤，应同步更新本文件中的状态、测试结果和遗留问题，确保比赛现场能够快速判断当前固件具备哪些能力。

状态定义：

- `已完成`：代码、编译和对应硬件测试均已完成。
- `进行中`：已经开始设计或实现，但尚未通过全部验收测试。
- `待开始`：尚未进入实现。
- `阻塞`：存在明确外部条件或前置问题，暂时无法继续。

## 2. 总体目标

最终控制系统应支持：

- 稳定控制左右轮转速，并在通信异常时可靠停车。
- 使用 GY931 航向角实现直行航向保持和相对角度转弯闭环。
- 非阻塞执行直行、转弯、等待、停车和循迹等基础动作。
- 使用动作序列快速组合比赛流程，不需要为每条路线重新编写状态机。
- 通过 Shell 单独测试动作、修改参数、查看遥测并保存比赛参数。
- 在动作超时、传感器异常、通信错误或人工取消时立即进入安全停车状态。

## 3. 当前基线

### 3.1 已完成功能

| 项目 | 状态 | 当前结果 |
| --- | --- | --- |
| MotorDriver I2C 通信 | 已完成 | 主控通过 I2C 控制电机板 |
| I2C 写后校验 | 已完成 | 关键配置和运动寄存器写入后读回确认 |
| 运动指令续租 | 已完成 | 主控每 100 ms 刷新目标转速，避免电机板看门狗超时 |
| 续租不清零 PWM | 已完成 | 续租只写目标转速，不重复写模式和占空比 |
| 双轮命令失败保护 | 已完成 | 任一侧命令失败时停止两侧电机并清除活动命令 |
| Shell 串口迁移 | 已完成 | UART0，PB1/RX，PB0/TX，115200 8N1 |
| 编码器方向 | 已完成 | M1 输出和编码器反向，M2 保持正常方向 |
| 编码器 CPR | 已完成 | 左右轮均为 364 counts/rev |
| 地面速度 PID | 已完成 | Q4.4 原始值 Kp=1、Ki=1、Kd=0 |
| 输出限制 | 已完成 | min duty=6%，max duty=50% |
| 默认参数同步 | 已完成 | gugaPI 和 MotorDriver 源码默认值已统一 |

### 3.2 已完成地面测试

| 目标转速 | 实测结果 | 结论 |
| --- | --- | --- |
| 80 RPM | 稳态约 76～82 RPM | 能可靠起步，低速可用 |
| 150 RPM | 稳态约 138～155 RPM | 正反转可用 |
| 300 RPM | 稳态约 267～323 RPM | 稳态可用，阶跃起步存在约 380 RPM 超调 |

### 3.3 当前主要问题

1. MotorDriver 速度环周期为 100 ms，控制响应较粗。
2. PID 参数使用 Q4.4 的 8 位整数，最小非零增益为 1/16，无法进行更细调节。
3. 目标转速阶跃变化时，比例项和积分项同时作用，300 RPM 起步存在明显超调。
4. 当前缺少目标转速斜坡和速度前馈。
5. 当前尚未建立 GY931 航向角连续化、零点管理和闭环控制，无法稳定完成直行航向保持和按角度转弯。
6. 当前没有统一动作生命周期和动作序列执行器。

## 4. 阶段一：完善速度控制

状态：`进行中`

目标：在地面负载下稳定完成低速、高速和变速控制，为 GY931 航向闭环和动作控制提供可靠基础。

### 4.1 明确速度环行为

状态：`进行中`

当前进度：代码、编译、MotorDriver v2 烧录和主控 I2C 联调已完成；等待架空及地面变速测试。

实现内容：

- 整理速度模式进入、目标变化、方向变化和停车时的状态转换。
- 只有模式切换、方向切换、故障和停车时完整重置 PID。
- 普通目标转速变化时避免无条件清空全部控制状态。
- 明确积分限幅、输出限幅和低速最小占空比之间的关系。

测试：

- 连续发送相同目标，确认 PID 状态不会被重置。
- 执行 `80 -> 150 -> 300 -> 150 -> 80 RPM`，检查过程连续性。
- 正转切换反转时确认先停车或经过受控换向过程。

验收标准：

- 相同目标续租不影响 PWM 输出。
- 目标变化过程中不存在失控加速。
- 方向切换过程中不存在长时间反向制动电流。

### 4.2 加入目标转速斜坡

状态：`进行中`

当前进度：已实现 MotorDriver 内部请求目标与控制目标分离，默认加速 600 RPM/s、减速 900 RPM/s；运行内存已验证，等待运动验收。

实现内容：

- 在 MotorDriver 内区分请求目标和控制目标。
- 控制目标按固定加速度逐步接近请求目标。
- 分别配置加速斜率和减速斜率。
- 停车、急停和故障不受普通减速斜坡限制，应能立即停止输出。

初始建议参数：

- 加速斜率：400～800 RPM/s。
- 减速斜率：600～1200 RPM/s。
- 控制周期内使用整数余量累计，避免低斜率被整数截断。

测试：

- `0 -> 80 RPM` 低速起步。
- `0 -> 300 RPM` 中速起步。
- `80 -> 300 -> 80 RPM` 连续变速。
- 300 RPM 运行时执行普通停车和急停。

验收标准：

- 300 RPM 起步超调不超过 15%。
- 目标变化期间两轮速度差不持续扩大。
- 急停不被速度斜坡延迟。

### 4.3 增加速度环遥测

状态：`进行中`

当前进度：已增加控制目标、速度误差、积分状态和 PWM 遥测寄存器，并接入 `motor rpm`；静止状态 Shell 联调通过，等待运动遥测验收。

实现内容：

- 提供请求目标转速、斜坡目标转速和实测转速。
- 提供速度误差、积分状态和最终 PWM 占空比。
- Shell 支持单次查看和周期输出。
- 周期输出不得阻塞控制循环或造成 I2C 总线拥塞。

测试：

- 遥测关闭时不产生额外串口输出。
- 10 Hz 遥测时控制周期保持稳定。
- 遥测值与 MotorDriver 寄存器值一致。

验收标准：

- 能仅通过日志定位超调、饱和和起步失败原因。
- 遥测开启后速度波动没有明显增加。

### 4.4 速度环地面回归测试

状态：`待开始`

测试矩阵：

| 项目 | 测试点 |
| --- | --- |
| 恒速 | 60、80、150、300、500 RPM |
| 变速 | 80/150/300 RPM 相互切换 |
| 方向 | 正转、反转、受控换向 |
| 负载 | 架空、地面直行、轻微阻力 |
| 电源 | 满电和比赛允许的低电压状态 |
| 持续时间 | 1 秒阶跃、10 秒恒速、连续20次启停 |

验收标准：

- 稳态误差不超过目标的 10% 或 10 RPM，取较大值。
- 阶跃超调不超过 15%。
- 左右轮稳态差不超过 10%。
- 连续20次启停无通信错误、失控或无法起步。

## 5. 阶段二：GY931 航向闭环

状态：`待开始`

目标：使用 GY931 航向角实现直行航向保持和相对角度转弯，不实现底盘里程计、行驶距离估计或二维位姿。

### 5.1 航向采集与角度处理

实现内容：

- 周期读取 GY931 yaw，记录数据时间戳和通信状态。
- 将传感器角度转换为统一的毫度单位，并正确处理 `-180°/180°` 或 `0°/360°` 回绕。
- 提供最短角度差、相对航向和连续航向累计接口。
- 对静止抖动进行轻量滤波，但不得引入影响转弯停车的明显延迟。
- 记录最后有效值、连续失败次数和数据是否过期。

测试：

- 静止连续采样，统计零点波动和长期漂移。
- 手动跨越角度回绕点，确认角度差连续且方向正确。
- 左右转动车身，确认 yaw 正负方向约定一致。
- 电机停止和运行时分别采样，检查电磁干扰和通信稳定性。

验收标准：

- 静止短期波动可量化且不会导致控制输出持续抖动。
- 跨回绕点时不出现接近 360° 的错误跳变。
- 传感器读取失败或数据过期能够被控制层可靠识别。

### 5.2 直行航向保持

实现内容：

- 动作启动时锁定当前 yaw 为目标航向。
- 根据最短航向误差生成左右轮差速修正：`left = base - correction`，`right = base + correction`。
- 对修正量、单轮目标转速和误差积分进行限幅。
- 支持前进和倒车，明确倒车时修正方向。
- 基础速度过低、传感器无效或误差过大时进入安全停车，而不是继续盲目修正。

测试：

- `80、150、300 RPM` 定时直行，记录起始和结束 yaw。
- 人工轻推车身产生小角度扰动，确认能够回正且不振荡。
- 前进和倒车分别测试左右偏差修正方向。
- 连续10次相同动作，统计最终航向误差。

验收标准：

- 直行结束航向误差初期目标不超过 `5°`，调试稳定后目标不超过 `3°`。
- 左右修正输出不持续饱和，车身不出现蛇形振荡。
- GY931 故障、超时或数据过期时立即停止底盘。

### 5.3 相对角度转弯控制

实现内容：

- 支持相对左转和右转，目标角度至少覆盖 `±180°`，并正确处理回绕。
- 使用剩余角度生成原地转向或差速转弯速度，接近目标时分段减速。
- 使用角度容差和连续稳定时间判定完成，避免单次采样越界即结束。
- 提供最大转速、最小转速、减速角度、角度容差、稳定时间和超时参数。
- 超调后允许受限反向修正，超过最大修正次数则故障停车。

测试：

- 原地左转、右转 `30°、45°、90°、180°`。
- 跨 yaw 回绕点执行左右转弯。
- 不同电池电压和地面阻力下重复测试。
- 执行中取消、GY931 断连和动作超时测试。

验收标准：

- `90°` 转向误差初期不超过 `5°`，调试稳定后目标不超过 `3°`。
- `180°` 转向误差不超过 `5°`。
- 左转和右转误差接近，不依赖仅对单一方向有效的补偿。
- 所有失败路径最终进入停车状态。

### 5.4 参数与遥测

实现内容：

- 航向滤波、直行控制和转弯控制参数进入 ConfigStore。
- Shell 输出目标 yaw、当前 yaw、连续 yaw、航向误差、修正量和传感器新鲜度。
- 提供单次状态查看和可关闭的周期 CSV 遥测。
- 比赛参数必须显式保存，调试参数不得自动覆盖默认值。

## 6. 阶段三：非阻塞基础动作

状态：`待开始`

目标：所有动作都通过周期更新执行，运行过程中 Shell、传感器、看门狗和急停仍可工作。

### 6.1 统一动作接口

动作生命周期：

```text
Idle -> Running -> Success
                -> Timeout
                -> Fault
                -> Cancelled
```

统一能力：

- 启动动作。
- 周期更新动作。
- 查询状态和进度。
- 取消动作。
- 超时处理。
- 安全停车。

验收标准：

- 任意动作均不能阻塞主循环。
- 动作失败、超时和取消都必须停止底盘。
- 同一动作对象完成后可以重新启动。

### 6.2 定时速度动作

实现内容：

- 指定左右轮 RPM 和持续时间。
- 支持直行、倒车、原地转向和弧线运动。
- 作为最简单的 ActionRunner 联调动作。

测试：

- 正常完成。
- 执行中取消。
- 超时和通信故障。

### 6.3 定时航向直行动作

实现内容：

- 输入基础 RPM、持续时间、航向控制参数和超时时间。
- 动作启动时锁定当前 GY931 yaw。
- 运行期间使用航向误差修正左右轮目标。
- 到达持续时间后停车，不承诺精确行驶距离。

验收标准：

- 动作持续时间误差满足调度周期要求。
- 结束航向误差不超过阶段二的直行验收标准。
- 连续10次动作无明显航向发散趋势。

### 6.4 按角度转弯动作

实现内容：

- 支持原地左转、原地右转和指定半径转弯。
- 根据 GY931 目标航向和剩余角度减速。
- 使用角度容差和稳定时间判定动作完成。

验收标准：

- 90°转向误差不超过 3°～5°。
- 180°转向误差不超过 5°。
- 左转和右转误差接近，不依赖单向补偿。

### 6.5 等待和停车动作

实现内容：

- 非阻塞等待指定时间。
- 普通停车等待速度降到阈值。
- 急停立即清除动作和运动租约。

## 7. 阶段四：动作序列 ActionRunner

状态：`待开始`

目标：通过动作表快速组合比赛流程。

### 7.1 序列执行器

实现内容：

- 顺序启动和更新动作。
- 当前步骤成功后自动进入下一步。
- 任一步骤失败、超时或取消时停止整个序列。
- 提供序列状态、当前步骤、总步骤数和错误原因。

示例流程：

```text
定时直行 1500 ms（GY931 航向保持）
等待 100 ms
左转 90°
定时直行 1000 ms（GY931 航向保持）
开始循迹
检测到终点后停车
```

测试：

- 空序列。
- 单动作序列。
- 多动作正常完成。
- 中间步骤超时。
- 执行中取消。
- 完成后再次启动。

### 7.2 Shell 调试接口

实现内容：

- 查看所有可用动作和动作序列。
- 单独启动任意动作。
- 从指定步骤开始运行。
- 查看当前状态、进度和错误。
- 暂停、继续、取消和急停。

验收标准：

- 比赛调车时无需重新编译即可单独验证一个动作。
- Shell 命令错误不会启动意外动作。

## 8. 阶段五：循迹动作

状态：`待开始`

目标：把灰度传感器处理、循迹控制和赛道事件识别封装成可组合动作。

### 8.1 灰度传感器标定

实现内容：

- 保存各通道黑白标定值。
- 归一化各通道读数。
- 计算线路位置、置信度和丢线状态。
- 标定参数保存到 FRAM。

测试：

- 白底、黑线、边缘位置和环境光变化。
- 手持传感器左右移动，确认误差方向连续且正确。

### 8.2 基础循迹控制

控制形式：

```text
left_rpm  = base_rpm - correction
right_rpm = base_rpm + correction
```

实现内容：

- 线路误差控制器。
- 输出限幅和速度降额。
- 根据弯道误差动态调整基础速度。
- 丢线时保留最后方向并限制搜索时间。

测试顺序：

1. 低速直线。
2. 缓弯。
3. 连续 S 弯。
4. 急弯。
5. 短暂丢线和恢复。

### 8.3 赛道事件识别

实现内容：

- 十字、直角、起点和终点识别。
- 事件去抖和最小间隔限制。
- 事件触发后由 ActionRunner 决定下一动作。

验收标准：

- 同一标志不会重复触发。
- 普通宽线和弯道不会误判为终点。
- 识别失败时能够超时停车。

## 9. 阶段六：比赛调车与参数管理

状态：`待开始`

### 9.1 参数体系

实现内容：

- 速度环、GY931 航向闭环、动作和循迹参数统一进入 ConfigStore。
- 区分默认值、开发值和比赛值。
- 参数变更后显示 dirty 状态，显式执行保存。
- 配置结构升级时提供版本和兼容策略。

### 9.2 调车遥测

实现内容：

- 支持按固定频率输出 CSV 格式数据。
- 输出时间、动作状态、目标速度、实际速度、目标航向、当前航向和航向误差。
- 支持快速关闭遥测，避免比赛模式串口负载。

### 9.3 比赛模式

实现内容：

- 上电后保持安全静止。
- 按键或明确命令启动比赛序列。
- 关闭非必要日志和诊断任务。
- 故障后锁定停车，必须明确复位才能重新启动。

## 10. 系统级回归测试

状态：`待开始`

必须覆盖：

- 主控与 MotorDriver 分别重启。
- I2C 短时错误和持续错误。
- 编码器断线、方向错误和异常跳变。
- FRAM 加载失败和恢复默认值。
- 低电压下起步、转弯和停车。
- 连续运行完整比赛流程至少20次。
- Shell 调试模式和比赛模式分别构建。

最终验收标准：

- 完整流程成功率达到比赛要求。
- 任意单点通信故障不会导致电机持续失控。
- 所有失败路径最终都能进入停车状态。
- 参数、固件版本和测试结果有明确记录。

## 11. 实施顺序

按以下顺序推进，前一阶段未通过验收前，不把后一阶段作为比赛主流程：

1. 完善速度控制状态机和目标斜坡。
2. 增加速度环遥测并完成地面回归。
3. 实现 GY931 航向采集、直行保持和相对角度转弯闭环。
4. 实现统一动作接口。
