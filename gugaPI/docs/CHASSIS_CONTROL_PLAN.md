# 小车控制开发计划

## 1. 文档目的

本文档用于跟踪电赛小车控制功能的设计、实现、测试和验收状态。
每完成一个实现步骤，应同步更新本文件中的状态、测试结果和遗留问题，确保比赛现场能够快速判断当前固件具备哪些能力。

状态定义：

- `已完成`：代码、编译和对应硬件测试均已完成。
- `进行中`：已经开始设计或实现，但尚未通过全部验收测试。
- `待开始`：尚未进入实现。
- `阻塞`：存在明确外部条件或前置问题，暂时无法继续。

## 2. 总体目标

最终控制系统应支持：

- 稳定控制左右轮转速，并在通信异常时可靠停车。
- 根据编码器估计行驶距离、航向角和二维位姿。
- 非阻塞执行直行、转弯、等待、停车和循迹等基础动作。
- 使用动作序列快速组合比赛流程，不需要为每条路线重新编写状态机。
- 通过 Shell 单独测试动作、修改参数、查看遥测并保存比赛参数。
- 在动作超时、传感器异常、通信错误或人工取消时立即进入安全停车状态。

## 3. 当前基线

### 3.1 已完成功能

| 项目 | 状态 | 当前结果 |
| --- | --- | --- |
| MotorDriver I2C 通信 | 已完成 | 主控通过 I2C 控制电机板 |
| I2C 写后校验 | 已完成 | 关键配置和运动寄存器写入后读回确认 |
| 运动指令续租 | 已完成 | 主控每 100 ms 刷新目标转速，避免电机板看门狗超时 |
| 续租不清零 PWM | 已完成 | 续租只写目标转速，不重复写模式和占空比 |
| 双轮命令失败保护 | 已完成 | 任一侧命令失败时停止两侧电机并清除活动命令 |
| Shell 串口迁移 | 已完成 | UART0，PB1/RX，PB0/TX，115200 8N1 |
| 编码器方向 | 已完成 | M1 输出和编码器反向，M2 保持正常方向 |
| 编码器 CPR | 已完成 | 左右轮均为 364 counts/rev |
| 地面速度 PID | 已完成 | Q4.4 原始值 Kp=1、Ki=1、Kd=0 |
| 输出限制 | 已完成 | min duty=6%，max duty=50% |
| 默认参数同步 | 已完成 | gugaPI 和 MotorDriver 源码默认值已统一 |

### 3.2 已完成地面测试

| 目标转速 | 实测结果 | 结论 |
| --- | --- | --- |
| 80 RPM | 稳态约 76～82 RPM | 能可靠起步，低速可用 |
| 150 RPM | 稳态约 138～155 RPM | 正反转可用 |
| 300 RPM | 稳态约 267～323 RPM | 稳态可用，阶跃起步存在约 380 RPM 超调 |

### 3.3 当前主要问题

1. MotorDriver 速度环周期为 100 ms，控制响应较粗。
2. PID 参数使用 Q4.4 的 8 位整数，最小非零增益为 1/16，无法进行更细调节。
3. 目标转速阶跃变化时，比例项和积分项同时作用，300 RPM 起步存在明显超调。
4. 当前缺少目标转速斜坡和速度前馈。
5. 当前没有底盘里程计，无法按距离或角度闭环完成动作。
6. 当前没有统一动作生命周期和动作序列执行器。

## 4. 阶段一：完善速度控制

状态：`进行中`

目标：在地面负载下稳定完成低速、高速和变速控制，为里程计和动作控制提供可靠基础。

### 4.1 明确速度环行为

状态：`待开始`

实现内容：

- 整理速度模式进入、目标变化、方向变化和停车时的状态转换。
- 只有模式切换、方向切换、故障和停车时完整重置 PID。
- 普通目标转速变化时避免无条件清空全部控制状态。
- 明确积分限幅、输出限幅和低速最小占空比之间的关系。

测试：

- 连续发送相同目标，确认 PID 状态不会被重置。
- 执行 `80 -> 150 -> 300 -> 150 -> 80 RPM`，检查过程连续性。
- 正转切换反转时确认先停车或经过受控换向过程。

验收标准：

- 相同目标续租不影响 PWM 输出。
- 目标变化过程中不存在失控加速。
- 方向切换过程中不存在长时间反向制动电流。

### 4.2 加入目标转速斜坡

状态：`待开始`

实现内容：

- 在 MotorDriver 内区分请求目标和控制目标。
- 控制目标按固定加速度逐步接近请求目标。
- 分别配置加速斜率和减速斜率。
- 停车、急停和故障不受普通减速斜坡限制，应能立即停止输出。

初始建议参数：

- 加速斜率：400～800 RPM/s。
- 减速斜率：600～1200 RPM/s。
- 控制周期内使用整数余量累计，避免低斜率被整数截断。

测试：

- `0 -> 80 RPM` 低速起步。
- `0 -> 300 RPM` 中速起步。
- `80 -> 300 -> 80 RPM` 连续变速。
- 300 RPM 运行时执行普通停车和急停。

验收标准：

- 300 RPM 起步超调不超过 15%。
- 目标变化期间两轮速度差不持续扩大。
- 急停不被速度斜坡延迟。

### 4.3 增加速度环遥测

状态：`待开始`

实现内容：

- 提供请求目标转速、斜坡目标转速和实测转速。
- 提供速度误差、积分状态和最终 PWM 占空比。
- Shell 支持单次查看和周期输出。
- 周期输出不得阻塞控制循环或造成 I2C 总线拥塞。

测试：

- 遥测关闭时不产生额外串口输出。
- 10 Hz 遥测时控制周期保持稳定。
- 遥测值与 MotorDriver 寄存器值一致。

验收标准：

- 能仅通过日志定位超调、饱和和起步失败原因。
- 遥测开启后速度波动没有明显增加。

### 4.4 速度环地面回归测试

状态：`待开始`

测试矩阵：

| 项目 | 测试点 |
| --- | --- |
| 恒速 | 60、80、150、300、500 RPM |
| 变速 | 80/150/300 RPM 相互切换 |
| 方向 | 正转、反转、受控换向 |
| 负载 | 架空、地面直行、轻微阻力 |
| 电源 | 满电和比赛允许的低电压状态 |
| 持续时间 | 1 秒阶跃、10 秒恒速、连续20次启停 |

验收标准：

- 稳态误差不超过目标的 10% 或 10 RPM，取较大值。
- 阶跃超调不超过 15%。
- 左右轮稳态差不超过 10%。
- 连续20次启停无通信错误、失控或无法起步。

## 5. 阶段二：底盘里程计

状态：`待开始`

目标：使用左右编码器计算小车行驶距离、航向变化和二维位姿。

### 5.1 编码器增量与轮距计算

实现内容：

- 保存左右编码器上次计数并计算有符号增量。
- 根据 CPR 和轮径换算左右轮距离。
- 处理编码器计数溢出。
- 提供累计左右轮距离和底盘中心距离。

测试：

- 手动转动单轮，确认距离方向正确。
- 左右轮转动一圈，确认计算距离接近轮子周长。
- 正向一圈再反向一圈，累计增量接近零。

### 5.2 位姿估计

实现内容：

- 根据差速底盘模型计算 `x、y、heading`。
- 航向角内部使用统一单位，避免角度和弧度混用。
- 提供里程计清零和设置初始位姿接口。
- 为后续 IMU 融合保留航向修正入口。

测试：

- 直行 500 mm 和 1000 mm。
- 原地旋转 90°、180°和360°。
- 前进后原路返回。
- 连续执行方形路线。

验收标准：

- 1000 mm 直行距离误差不超过 5%。
- 原地旋转 90°误差不超过 5°。
- 同一动作连续10次结果无持续发散趋势。

### 5.3 几何参数标定

实现内容：

- 标定左右轮有效轮径。
- 标定有效轮距，而不是只使用机械图纸值。
- 如左右轮存在系统差异，增加左右距离比例系数。
- 参数通过 ConfigStore 保存到 FRAM。

测试：

- 多次长距离直行标定轮径。
- 多次360°旋转标定轮距。
- 正反向分别验证参数，避免只补偿单一方向。

## 6. 阶段三：非阻塞基础动作

状态：`待开始`

目标：所有动作都通过周期更新执行，运行过程中 Shell、传感器、看门狗和急停仍可工作。

### 6.1 统一动作接口

动作生命周期：

```text
Idle -> Running -> Success
                -> Timeout
                -> Fault
                -> Cancelled
```

统一能力：

- 启动动作。
- 周期更新动作。
- 查询状态和进度。
- 取消动作。
- 超时处理。
- 安全停车。

验收标准：

- 任意动作均不能阻塞主循环。
- 动作失败、超时和取消都必须停止底盘。
- 同一动作对象完成后可以重新启动。

### 6.2 定时速度动作

实现内容：

- 指定左右轮 RPM 和持续时间。
- 支持直行、倒车、原地转向和弧线运动。
- 作为最简单的 ActionRunner 联调动作。

测试：

- 正常完成。
- 执行中取消。
- 超时和通信故障。

### 6.3 按距离直行动作

实现内容：

- 输入目标距离、最大速度、加减速参数和超时时间。
- 根据剩余距离生成速度目标。
- 使用左右轮距离差进行直行修正。
- 接近终点时减速并进入距离容差。

验收标准：

- 500 mm 直行误差不超过 20 mm。
- 1000 mm 直行误差不超过 30 mm。
- 连续10次动作无明显累积偏向。

### 6.4 按角度转弯动作

实现内容：

- 支持原地左转、原地右转和指定半径转弯。
- 根据目标角度和剩余角度减速。
- 先使用编码器航向，后续可切换到 IMU 融合航向。

验收标准：

- 90°转向误差不超过 3°～5°。
- 180°转向误差不超过 5°。
- 左转和右转误差接近，不依赖单向补偿。

### 6.5 等待和停车动作

实现内容：

- 非阻塞等待指定时间。
- 普通停车等待速度降到阈值。
- 急停立即清除动作和运动租约。

## 7. 阶段四：动作序列 ActionRunner

状态：`待开始`

目标：通过动作表快速组合比赛流程。

### 7.1 序列执行器

实现内容：

- 顺序启动和更新动作。
- 当前步骤成功后自动进入下一步。
- 任一步骤失败、超时或取消时停止整个序列。
- 提供序列状态、当前步骤、总步骤数和错误原因。

示例流程：

```text
直行 500 mm
等待 100 ms
左转 90°
直行 300 mm
开始循迹
检测到终点后停车
```

测试：

- 空序列。
- 单动作序列。
- 多动作正常完成。
- 中间步骤超时。
- 执行中取消。
- 完成后再次启动。

### 7.2 Shell 调试接口

实现内容：

- 查看所有可用动作和动作序列。
- 单独启动任意动作。
- 从指定步骤开始运行。
- 查看当前状态、进度和错误。
- 暂停、继续、取消和急停。

验收标准：

- 比赛调车时无需重新编译即可单独验证一个动作。
- Shell 命令错误不会启动意外动作。

## 8. 阶段五：循迹动作

状态：`待开始`

目标：把灰度传感器处理、循迹控制和赛道事件识别封装成可组合动作。

### 8.1 灰度传感器标定

实现内容：

- 保存各通道黑白标定值。
- 归一化各通道读数。
- 计算线路位置、置信度和丢线状态。
- 标定参数保存到 FRAM。

测试：

- 白底、黑线、边缘位置和环境光变化。
- 手持传感器左右移动，确认误差方向连续且正确。

### 8.2 基础循迹控制

控制形式：

```text
left_rpm  = base_rpm - correction
right_rpm = base_rpm + correction
```

实现内容：

- 线路误差控制器。
- 输出限幅和速度降额。
- 根据弯道误差动态调整基础速度。
- 丢线时保留最后方向并限制搜索时间。

测试顺序：

1. 低速直线。
2. 缓弯。
3. 连续 S 弯。
4. 急弯。
5. 短暂丢线和恢复。

### 8.3 赛道事件识别

实现内容：

- 十字、直角、起点和终点识别。
- 事件去抖和最小间隔限制。
- 事件触发后由 ActionRunner 决定下一动作。

验收标准：

- 同一标志不会重复触发。
- 普通宽线和弯道不会误判为终点。
- 识别失败时能够超时停车。

## 9. 阶段六：比赛调车与参数管理

状态：`待开始`

### 9.1 参数体系

实现内容：

- 速度环、里程计、动作和循迹参数统一进入 ConfigStore。
- 区分默认值、开发值和比赛值。
- 参数变更后显示 dirty 状态，显式执行保存。
- 配置结构升级时提供版本和兼容策略。

### 9.2 调车遥测

实现内容：

- 支持按固定频率输出 CSV 格式数据。
- 输出时间、动作状态、目标速度、实际速度、位姿和传感器误差。
- 支持快速关闭遥测，避免比赛模式串口负载。

### 9.3 比赛模式

实现内容：

- 上电后保持安全静止。
- 按键或明确命令启动比赛序列。
- 关闭非必要日志和诊断任务。
- 故障后锁定停车，必须明确复位才能重新启动。

## 10. 系统级回归测试

状态：`待开始`

必须覆盖：

- 主控与 MotorDriver 分别重启。
- I2C 短时错误和持续错误。
- 编码器断线、方向错误和异常跳变。
- FRAM 加载失败和恢复默认值。
- 低电压下起步、转弯和停车。
- 连续运行完整比赛流程至少20次。
- Shell 调试模式和比赛模式分别构建。

最终验收标准：

- 完整流程成功率达到比赛要求。
- 任意单点通信故障不会导致电机持续失控。
- 所有失败路径最终都能进入停车状态。
- 参数、固件版本和测试结果有明确记录。

## 11. 实施顺序

按以下顺序推进，前一阶段未通过验收前，不把后一阶段作为比赛主流程：

1. 完善速度控制状态机和目标斜坡。
2. 增加速度环遥测并完成地面回归。
3. 实现编码器里程计和几何参数标定。
4. 实现统一动作接口。
5. 实现定时、直行、转弯、等待和停车动作。
6. 实现 ActionRunner 和 Shell 调试命令。
7. 实现灰度标定、循迹和赛道事件识别。
8. 完成参数持久化、比赛模式和系统级回归。

## 13. 下一步

下一项具体任务：设计并实现 MotorDriver 内部目标转速斜坡，同时修正目标变化时的 PID 状态处理。

开始编码前需要先确定：

- 目标加速和减速斜率的配置单位。
- 普通停车与急停的行为差异。
- 目标变化时积分项是保留、限幅还是条件清零。
- 需要增加哪些寄存器和 Shell 遥测字段。
