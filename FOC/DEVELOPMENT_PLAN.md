# FOC 实时性与可靠性改进计划

## 目标

在保留现有空载验证结果的基础上，把当前调试原型逐步改造成时序可测、
参数可配置、故障可追踪，并能够安全开展带载和高速验证的电机控制固件。

本计划遵循以下原则：

- 每一步都保持上电默认不转动。
- 实时控制路径不等待 UART、CAN 或 I2C。
- 先建立测量手段，再提高环路带宽和功率限制。
- 每次控制算法修改都先空载、低母线限流验证，再进入带载测试。
- `empty.syscfg` 是外设配置源；不手工维护 `Debug/ti_msp_dl_config.*`。

## 阶段 1：建立确定性的实时基础

### 1.1 非阻塞 UART

- [x] 用 TX 环形缓冲区替代逐字节阻塞发送。
- [x] 使用主循环协作式 FIFO 服务排空缓冲区，不等待单字节发送完成。
- [x] 缓冲区满时丢弃新数据并记录计数，控制路径不得等待。
- [x] 增加 `u` 命令显示 UART 排队字节数、容量和丢弃计数。
- [x] COM4 实机确认启动信息和 `f` 长状态行完整，前后 `u` 均为 `dropped=0`。

验收条件：运行 FOC 时打印最长状态行不会暂停主循环读取编码器；编译无警告。

### 1.2 编码器固定周期异步采样

- [x] 将运行时 AS5048B 快速读取改为协作式非阻塞 I2C 状态机。
- [x] 以 20 kHz ADC 序号为时间基准，每 20 tick 发起一次采样（1 kHz）。
- [x] 记录请求、成功、错误、超时、漏周期和最大完成间隔。
- [x] 控制运行时禁止完整诊断读取抢占快速角度事务。

验收条件：角度采样周期和最坏间隔可测；UART/CAN 活动不改变采样周期。

COM4 静态验收结果：连续 `17888` 次成功事务，错误、超时和漏周期均为 0；
无诊断抢占时最大完成间隔为 `1000 us`。STOP 状态执行一次完整 `a` 诊断时，
快速状态机会暂停并自动恢复，实测最大间隔为 `1350 us`。`e` 命令用于查看
快速采样器统计。STOP 主循环使用 `WFI`，由 20 kHz ADC 中断唤醒协作服务。

### 1.3 电角度预测

- [x] 基于最近角度、ADC 时间戳和滤波电角速度预测当前电角度。
- [x] 在 20 kHz 电流环中使用预测电角度。
- [x] 单次预测最多外推 2 ms，避免长时间失联时无限外推。
- [x] 编码器失联阈值由 50 ms 收紧到 5 ms，即五个快速采样周期。

验收条件：在目标最高转速下，预测跨度和最大电角度误差有明确上限。

静态验收：COM4 下采样器连续 `13608` 次成功、零错误；静止时样本电角度
与预测电角度一致，滤波电角速度为 0。动态误差仍需在电机低速、反转和
目标最高转速下使用示波器/遥测验证，不能仅凭静态结果关闭本阶段。

25 RPM 动态验收：完成多轮对齐、启动和停止，无 nFAULT、过流或编码器失联；
最终状态采样为 `24 RPM` 和 `30 RPM`。运行统计累计 `205750` 次 I2C 成功、
零错误、零超时，最大间隔 `2.2 ms`，低于 `5 ms` 失联阈值。曾尝试把速度
反馈窗口延长到 100 ms，但反馈延迟造成 5～67 RPM 振荡，已回退；当前保留
10 ms 控制窗口，并移除了恒定目标下因瞬时超速而清空速度积分的逻辑。

后续 COM4 动态整定发现整数 IIR 使用 `(target-current)/8` 时，余差小于 8 会因
向零截断而永久停留，静止轴的速度反馈曾锁在 `+5/-5 RPM`，电角速度预测也会
留下非零残差。现改为带最小 1 count 收敛步长的整数滤波，机械速度和电角速度
都能真正回到零。速度环比例项由 `error/2` 提高为 `error`，25 RPM 的 1 秒间隔
瞬时读数由约 `3～47 RPM` 收敛到 `20～31 RPM`，末值为 `25 RPM`。

最终空载回归：+75 RPM 稳态为 `74～76 RPM`；从 +75 RPM 反转后约 1～2 秒
进入 `-73～-76 RPM`，反转瞬间 q 轴参考达到保守限幅进行主动制动。每次重新
开始对齐会恢复安全默认目标 `+25 RPM`，不再继承上一次运行的速度目标。

### 1.4 ISR 时序测量

- [x] 使用 32 MHz 电流采样定时器测量 ADC ISR 函数体执行时间。
- [x] 记录 ISR 最近/最大 tick 和超过 75% 周期预算的次数。
- [x] 增加 ADC 更新丢失、双 ADC 配对、溢出和中断重入的独立检测。
- [x] 将电流平均值、峰值等非保护统计降频，保护检测仍保持 20 kHz。

验收条件：最坏 ADC ISR 时间显著小于 50 us，并为更高优先级故障响应保留裕量。

STOP 静态基线：`j` 命令实测最大 `313` 个 32 MHz tick，约 `9.78 us`；
75% 周期预算为 `1200 tick`，超预算次数为 0。该方法测量 ISR 函数体，不包含
中断入口延迟；STOP 状态也不会执行完整 FOC 运算，因此仍需在低速和最高目标
转速运行时重新记录最大值。

动态回归基线：完整 FOC ISR 最大 `875 tick`，约 `27.34 us`，占 50 us
周期约 54%，75% 预算超限次数为 0。仍需增加入口延迟和 ADC 更新丢失检测，
并在更高目标转速、CAN 高负载和连续遥测条件下复测。

诊断统计现每 4 个电流样本取 1 点，以 64 点组成 12.8 ms 窗口；平均值和诊断
峰值的运算频率由 20 kHz 降到 5 kHz。约 600 mA 软件过流判断仍由电流环对每个
20 kHz 样本执行，不受诊断降频影响。

COM4 回归中，非统计周期 ISR 最近值为 `783～796 tick`，相比降频前常见的
`856～866 tick` 约减少 2 us；包含统计发布等最坏分支的累计峰值为 `983 tick`
（约 `30.72 us`），预算超限仍为 0。25/75 RPM 状态采样为 30/74 RPM，`i`
平均值和诊断峰值正常更新；累计超过 `1676` 万组 ADC 样本时同步错误仍全为 0。

### 1.5 PWM 同步更新和 ADC 一致性

- [x] 将三相比较值改为计数器零点统一更新，避免周期中途立即更新。
- [x] 验证 ADC0/ADC1 完成关系，不在 ADC1 结果未就绪时读取。
- [ ] 用示波器验证三相 PWM 对称性、采样点和极端占空比波形。

验收条件：三相比较值同一事件生效；采样发生在已验证的有效电流窗口。

固件与 COM4 验收：TIMA0 CC1/CC2/CC3 使用 `ZERO_EVT` 影子装载。ADC0 MEM1
中断内先检查 ADC1 MEM0 完成标志；三相未配对时跳过该次电流环，避免把新 A/B
与旧 C 相组合。`j` 同时统计 ADC0/ADC1 结果溢出、触发溢出、下溢、配对缺失、
异常 IRQ 和 ISR 重入。累计超过 `1068` 万组三相样本后上述错误计数均为 0。
完整 FOC ISR 峰值为 `961 tick`（约 `30.03 us`），75% 预算超限为 0。

动态回归：25 RPM 状态采样为 29 RPM，75 RPM 为 78 RPM；位置 `+10°` 后
误差约 0.4°，返回后误差 0.0°，全程无 nFAULT、过流或编码器失联。示波器
对三相边沿、零点装载时刻和 ADC 采样窗口的物理确认仍未完成，因此 1.5 尚未
完全关闭。

### 2.3 速度与位置控制

- [ ] 改进低速速度估算，评估自适应窗口或角度 PLL。
- [x] 修复整数速度/电角速度滤波无法衰减到零的问题。
- [x] 增加位置目标近区的限流 PD 电流控制，避免速度 PI 积分来回拉动。
- [ ] 增加速度、加速度和减速度限制。
- [ ] 增加多圈位置累计和受约束的位置轨迹。

验收条件：完成低速、额定工作点、反转、阶跃和带载扰动测试，并保存数据。

当前空载位置阶跃结果：目标 `+10°` 后稳态误差约 `0.4～0.7°`，返回原目标后
稳态误差约 `0.5～0.7°`，静止速度反馈为 `0 RPM`、偶发 `±1 RPM`。近区范围
为 128 encoder counts（约 2.81°），q 轴参考限幅约 ±64 mA。该结果尚未覆盖
外部负载、温升、母线变化和机械扰动，不能替代带载整定。

## 当前迭代

当前已完成实时通信基础、电角度预测、ISR 时序测量，以及第一轮空载速度/位置
控制改进。PWM 同步更新、ADC 更新丢失检测和带载参数整定仍是下一阶段重点。

当前结果：

- TI Arm Clang 5.1.1 LTS、`-O2 -Wall` 编译和链接通过，无警告。
- `Debug/FOC.out` 已重新构建并通过 DSLite program verification。
- 已烧录至 MSPM0G3507 并通过 COM4、115200 8N1 验证；电机保持 STOP，
  驱动、电流 ADC 和 AS5048B 初始化正常。
- 最终速度与位置阶跃回归无 nFAULT、过流或编码器失联；编码器统计零错误/
  零超时。启用 PWM 同步装载和 ADC 配对诊断后，ADC ISR 峰值 `961 tick`，
  超预算次数为 0。

实机调试记录：第一版 TX FIFO 中断实现虽然通过编译，但在 COM4 上产生乱码；
用旧固件对照确认 COM4 和 115200 8N1 硬件链路正常。因此撤回 TX 中断方案，
改为协作式非阻塞 FIFO 服务。后续如果主循环中仍存在不可接受的发送抖动，
直接进入 UART DMA，不再依赖当前芯片上的 TX 阈值中断路径。
